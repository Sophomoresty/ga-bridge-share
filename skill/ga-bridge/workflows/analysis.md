# Analysis Workflow

## 触发条件

- 任务以只读分析为主:
  - 当前网页结构
  - 已有文件集
  - 桌面状态
  - UI 现状
  - OCR / 抓取 / 证据整理

## 步骤

1. 先整理最小必要输入:
   - 绝对路径
   - 明确网址
   - 明确窗口或目标文件
2. 首轮分析直接启动并 watch:

```bash
ga start --task "..."
ga summary --job <job_id>
```

- 首次启动只用 `ga start ...`.
- 不要拆出第二个公开跟随命令.

3. 若首轮证据不够:

```bash
ga revise-job --job <job_id> --feedback "继续补这几项证据: 1. ... 2. ... 3. ..."
```

## 边界

- 分析默认只读, 不把写文件, 覆盖页面, 执行改动混进同一轮.
- 若下一步要落地修改, 回到对应业务 workflow 再开新轮.
